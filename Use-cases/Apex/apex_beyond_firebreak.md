# Apex — Beyond Firebreak Modules

## Purpose
Extend Apex beyond firebreak diagnostics to full waste-hunting capability across all three businesses.

---

### Core Waste-Hunting Modules

1. **Keyword Clustering & Intent Matching**  
   - Build clusters from full keyword universe.  
   - Match clusters to intent scores.  
   - Output: priority ranking of profitable vs wasteful clusters.

2. **SQR Cleaning & Negatives Automation**  
   - Daily SQR parsing at scale.  
   - Generate negative keyword lists by cluster (e.g. Kenya/South Africa leakage).  
   - Output: "£X spend excluded via automated negatives."  

3. **Landing Page Match Automation**  
   - Check ad → query → LP intent alignment.  
   - Output: "Botswana generic traffic misrouted to general safari LP, recommend redirect."  

4. **Geo Leakage Detector**  
   - Identify queries from non-core destinations.  
   - Output: "12% of UK Generic clicks contained Kenya/South Africa terms at £500–£1k CPL."  

5. **Funnel Drop-Off Mapping**  
   - Connect queries to booking curve progression.  
   - Detect clusters that drive leads but don’t progress.  
   - Output: "Cluster X generates 200 leads/month but <2 bookings → downweight."  

6. **Cross-Brand Query Shield**  
   - Synchronise learnings across Wilderness, Jacada, YZ.  
   - Waste flagged in one brand propagates to all.  
   - Output: "Term ‘cheap safari’ excluded group-wide after YZ flagged £900 CPL."  

---

## Operating Principle
- **CapitalCore:** Guardrails + capital allocation + efficiency narrative.  
- **Apex:** Waste hunting + query precision + operator empowerment.  

Together → dual system: CapitalCore protects capital, Apex hunts waste.
