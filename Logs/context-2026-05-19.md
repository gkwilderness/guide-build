### Post-verification issues found and fixed

The verification gate passed, but interactive testing revealed several issues from the migration that needed fixing:

#### 1. Container HOME was `/root` ‚Äî fixed
- **Cause:** `compose.yml` had `HOME: /root` hardcoded
- **Fix:** Changed to `HOME: /srv/openclaw` in `/srv/compose/openclaw.yml`
- **Why it mattered:** OpenClaw's skills watcher looks for `$HOME/.agents/skills` ‚Äî was hitting `/root` which uid=1002 (guide) can't access, causing repeated watcher errors in logs

#### 2. Container running as gareth (uid=1001) instead of guide (uid=1002) ‚Äî fixed
- **Cause:** Systemd override had `HOST_UID=$(id -u gareth)=1001`; all /srv/openclaw files are guide-owned from migration
- **Gareth updated** the systemd override to `HOST_UID=1002` (guide)
- Container now runs as uid=1002 gid=1004 (guide:guide-data) ‚Äî confirmed

#### 3. All channel agent auth-profiles.json were empty ‚Äî fixed
- **Cause:** Per-agent auth-profiles.json files all had `{"version": 1, "profiles": {}}` ‚Äî empty, no API key
- OpenClaw doesn't auto-populate from global credentials when auth-profiles.json exists but is empty
- **Fix:** Wrote valid auth-profiles to all 8 agents with the key from `/srv/openclaw/credentials/anthropic`
- Format: `{"version": 1, "profiles": {"anthropic:default": {"provider": "anthropic", "type": "api_key", "key": "..."}}}` 
- Main agent's old auth-profiles.json was guide-owned 600, inaccessible to gareth-running container ‚Äî deleted and rewrote

#### 4. TOOLS.md and BOOT.md had Mac Mini paths ‚Äî fixed
- Both files were migrated verbatim from the Mac Mini bundle with `~/` paths, Mac Mini hardware, old Tailscale URL
- Rewrote both to Z8 paths (`/srv/openclaw/workspaces/main/`, `/srv/guide-core/`, etc.)
- BOOT.md: updated to reflect Z8, removed false HubSpot data claim (vault not populated), noted OneDrive gap
- TOOLS.md: updated machine info (HP Z8 G4, Ubuntu, Docker), all `/srv/` paths, scripts noted at `.bak-20260512104802/` path

#### 5. tasks/ directory was guide-owned 700 ‚Äî fixed by Gareth (sudo)
- `[tasks/registry] Failed to restore task registry` on every boot
- **Gareth ran:** `sudo chown -R gareth:guide-data /srv/openclaw/tasks && sudo chmod -R 755 /srv/openclaw/tasks`
- Then superseded when container switched to guide uid ‚Äî tasks dir now accessible to guide

#### 6. Dead config file at /srv/openclaw/config/openclaw.json ‚Äî removed
