### 3. Backup script fixed

`/srv/guide-core/scripts/openclaw-backup.sh` was broken since the Mac Mini → Z8 migration. It was targeting `$HOME/.openclaw` (doesn't exist on Z8) and writing archives to `~/openclaw-backups/` (also wrong).

Fixes applied:
- Source: `$HOME/.openclaw` → `/srv/openclaw/`
- Destination: `~/openclaw-backups/` → `/srv/backup/dumps/`
- Runs as root via `sudo` — required because guide owns some files at `600` (auth tokens, etc.)
- NOPASSWD sudoers rule created at `/etc/sudoers.d/openclaw-backup`
- Crontab updated: `sudo /srv/guide-core/scripts/openclaw-backup.sh`
- Archive size: ~730MB → ~37MB by excluding agent session trajectories, npm dirs, `_inbox/` migration artefacts
- Exit code handling: treats tar exit 1 (skipped files) as warning, exit 2 as fatal

Commit: `guide-core` `324897a` — pushed to `github.com:gkwilderness/guide-core.git`

Verified: `sudo -n /srv/guide-core/scripts/openclaw-backup.sh` runs clean, archive at `/srv/backup/dumps/openclaw-20260527-1122.tar.gz` (37MB).
