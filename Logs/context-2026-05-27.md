# Context — 2026-05-27 (Engineer session)

## What this session was about

Infrastructure documentation and backup fix. No code changes to Guide itself — purely housekeeping.

Two goals:
1. Ensure every `/srv` directory has a CLAUDE.md so future sessions can orient quickly without needing to re-explore the machine.
2. Fix the OpenClaw backup script, which has been failing silently every night since the Mac Mini → Z8 migration in May.

---

## Decisions made

### CLAUDE.md coverage strategy

Wrote a top-level `/srv/CLAUDE.md` as the entry point (directory map, services table, critical rules). Then a per-directory CLAUDE.md for each subdirectory. Deferred heavily to existing docs (guide-core/CLAUDE.md, guide-build/CLAUDE.md) rather than duplicating. Placeholder directories (hermes, landing-pages, logs, onedrive, openwebui) got short files that explain their status and what activating them would require.

Also created `/srv/CRON.md` at the suggestion of the session. Rationale: all scheduled automation is in a single crontab and deserves one readable reference with context (status, blockers, env vars). Per-directory CRON.md files were considered and rejected — they'd scatter information that's best seen together.

### Backup script — run as root via sudo

The backup script needed to read `600`-mode files owned by the `guide` user (auth tokens, agent state). Options considered:
- `tar --ignore-failed-read` — skips unreadable files gracefully, but those files are worth having in the backup
- Root's crontab — would work, but gareth manages the crontab, not root
- NOPASSWD sudo rule for the specific script — chosen. Clean, auditable, least-privilege for this one script.

Added `/etc/sudoers.d/openclaw-backup` and updated crontab entry to `sudo /srv/guide-core/scripts/openclaw-backup.sh`.

### Archive size — what to exclude

Discovered the naive archive was 731MB. Traced large contributors:
- `agents/*/sessions/` — 750MB+ of conversation trajectory `.jsonl` files. Not needed for recovery (agent identity and memory are in `workspaces/`). Excluded.
- `_inbox/` — 670MB staging dir + Mac Mini migration artefacts. Excluded.
- `npm/` — 509MB npm packages. Reinstallable. Excluded.

Result: 37MB archive containing config, workspaces, credentials, skills, and agent auth state.

Note: `memory/` was excluded in the original Mac Mini script and remains excluded. This may be worth revisiting — agent memory is persistent knowledge, not just runtime state.
